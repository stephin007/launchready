Build a full-stack AI-powered web application called "LaunchReady" — a PRD and Sprint Planning tool built specifically for non-technical founders who need to communicate ideas to engineering teams.

---

## PRODUCT VISION
LaunchReady lets a non-technical founder answer 3 plain-English questions and get back a complete, engineering-ready Product Requirements Document with user stories, tasks, acceptance criteria, effort estimates, priorities, and sprint groupings — in 30 seconds.

Tagline: "Turn your idea into an engineering-ready plan in 30 seconds."

---

## TECH STACK
- Frontend: React (Vite)
- Backend: Node.js + Express
- Database: SQLite using better-sqlite3
- AI: OpenRouter API (NOT OpenAI directly)
  - Base URL: https://openrouter.ai/api/v1/chat/completions
  - Model: openai/gpt-4o
  - Auth header: Authorization: Bearer ${OPENROUTER_API_KEY}
  - Also send: HTTP-Referer: https://launchready.app and X-Title: LaunchReady
- Styling: Tailwind CSS + custom CSS variables from the design system below
- Icons: Lucide React
- Charts: Recharts
- Version Control: Git with minimum 6 commits, one per stage

## CORE USER FLOW

1. Founder lands on Home — bold hero, tagline, "Generate PRD" CTA
2. Fills in 3 plain-English questions (no jargon):
   - "What problem does your feature solve?"
   - "Who is this feature for?"
   - "What does success look like for this feature?"
3. Optional: product name field
4. Hits "Generate PRD" — animated loading state with status messages:
   - "Reading your idea..."
   - "Building user stories..."
   - "Allocating sprints..."
5. Gets back a fully structured PRD page with:
   - Product title + summary
   - Goals & success metrics
   - User stories (each with acceptance criteria)
   - Engineering tasks (linked to stories, with acceptance criteria)
   - Effort badges (S/M/L/XL)
   - Priority labels (P1/P2/P3 color coded)
   - Sprint groupings (Sprint 1, 2, 3 cards)
6. Can mark tasks as Todo / In Progress / Done (persisted to DB, shown inline)
7. Can copy a shareable link to this PRD (one click)
8. Can export PRD as Markdown (triggers browser download)
9. Every regeneration saves a version — viewable as a Change Log timeline

## BACKEND API ROUTES

POST /api/generate
- Accepts: { problem, audience, success, productName? }
- Calls OpenRouter API with structured prompt
- Runs output through Logic Layer
- Saves to SQLite
- Returns: full structured PRD object

GET /api/prds
- Returns all PRDs (for admin + history)

GET /api/prds/:id
- Returns single PRD by ID

GET /api/prds/share/:token
- Returns PRD by share token (public, read-only)

PATCH /api/prds/:id/tasks/:taskId
- Updates task status (todo/in-progress/done)

POST /api/prds/:id/regenerate
- Re-runs generation, saves new version to prd_versions

GET /api/admin/stats
- Returns: total PRDs, total tasks, avg effort score, PRDs per day (last 7), priority breakdown

## DATABASE SCHEMA (SQLite)

```sql
CREATE TABLE prds (
  id TEXT PRIMARY KEY,
  title TEXT NOT NULL,
  problem TEXT,
  audience TEXT,
  success_criteria TEXT,
  product_name TEXT,
  raw_output TEXT NOT NULL,
  share_token TEXT UNIQUE NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE task_status (
  id TEXT PRIMARY KEY,
  prd_id TEXT NOT NULL,
  task_id TEXT NOT NULL,
  status TEXT DEFAULT 'todo',
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (prd_id) REFERENCES prds(id)
);

CREATE TABLE prd_versions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  prd_id TEXT NOT NULL,
  version_number INTEGER NOT NULL,
  snapshot TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (prd_id) REFERENCES prds(id)
);
```
